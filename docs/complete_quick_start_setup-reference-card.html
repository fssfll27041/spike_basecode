Complete Dev Environment Setup Guide

Audience

Coaches — this guide assumes a fresh laptop with no prior robotics software installed.

This guide walks you through setting up a modern robotics coding workspace on your computer from scratch using MANUAL instructions. You will install a few free tools, download the team's shared code, and be ready to write and deploy programs to the robot. This is a one-time setup — once it's done, your everyday routine is just a few clicks. A separate, shorter guide covers the student workflow for laptops where setup is already complete.
NOTE: For easier setup, use this guide for automated scripts: Laptop Provisioning Guide

Phases 1 – 3 · Installation & Project Setup
1
Install Git
Git is how your computer downloads the team's shared code and tracks every change you make — like a save-history for your project. If something breaks, you can always go back.
Download from git-scm.com and accept all defaults. Verify in a terminal:
git --version
2
Install Python 3
Python is the language the robots are programmed in. Your computer needs it installed so you can run, test, and check code before it ever goes near the robot.
Download from python.org. During setup, check Add Python to PATH. Verify:
python --version
3
Install VS Code
VS Code is the editor where students write robot programs. It's free, widely used professionally, and has features that help catch mistakes while you type — before they cause problems on the field.
Download from code.visualstudio.com and install normally.
Project Setup
4
Download the Team's Code
This pulls a copy of the Bolton Robotics codebase from GitHub onto your computer. Every coach and student starts from the same shared foundation, so nobody is working in isolation.
Run in a terminal:
git clone https://github.com/fssfll/spike_basecode cd spike_basecode
The "spike_basecode" directory is the "root" of your repository
5
Create a Virtual Environment
This creates a private, self-contained workspace for this project's tools — keeping your robotics setup completely separate from anything else on your computer and ensuring every coach's setup behaves identically. Run from the root of your repository
python -m venv .venv
6
Activate the Virtual Environment
Activating "steps into" your private workspace. Do this once each time you open a new terminal. Your terminal prompt will show (.venv) as confirmation. Run from the root of your repository
# macOS / Linux source .venv/bin/activate # Windows .venv\Scripts\activate

Always activate first. If you skip this, commands run against your system Python instead of the project's tools, and things may not work as expected.

7
Install Project Tools
This installs everything the project depends on — including pybricksdev, the tool that sends your finished code to the robot over Bluetooth.
Run from the root of your repository AFTER activating the virtual environment.
pip install pybricks pybricksdev
Phases 4 – 5 · VS Code Setup & Daily Workflow
8
Open the Project in VS Code
Opening the whole project folder — not just a single file — lets VS Code see and apply all the project's settings and configuration at once.
File → Open Folder → select the spike_basecode folder.
9
Install VS Code Extensions
VS Code doesn't understand Python by default — extensions teach it the language. These three unlock error highlighting, autocomplete, and the ability to deploy code directly from the editor.
Open Extensions with Ctrl+Shift+X and install each:
ms-python.python · Python
ms-python.pylance · Pylance (autocomplete)
ms-python.debugpy · Python Debugger
ℹ️

About these extensions and the debugger. pybricksdev is a Python CLI tool that communicates with the robot over Bluetooth/BLE — the .venv provides a sandbox for it to run. ms-python.debugpy is installed because the ms-python.python extension requires it, but "noDebug": true in launch.json bypasses the debugger entirely. You cannot set breakpoints or attach to running code on the hub — the robot executes MicroPython on the SPIKE Prime, not on your laptop.

10
Point VS Code at Your Virtual Environment
VS Code needs to know which Python to use. Selecting your .venv activates autocomplete and error checking tuned specifically for Pybricks — the robot's programming library.
Press Ctrl+Shift+P → type Python: Select Interpreter → choose the option labeled .venv.

This is the payoff. After this step, VS Code highlights mistakes and offers completions as students type — catching errors before code ever touches the robot.

Phase 5 · Your Daily Routine

Once setup is complete, the heavy lifting is behind you. Every session follows this simple loop.

✏️
Write or edit code in VS Code
Autocomplete and error highlighting help students catch mistakes as they type, not after the robot fails on the field.
🔵
Power on the SPIKE Hub
The hub communicates with your computer over Bluetooth — no USB cable or extra app required.
▶️
Open the Run panel and deploy
Click the Run and Debug icon in the left sidebar (or press Ctrl+Shift+D). Use the dropdown at the top of the panel to select the robot you want to program — each robot on your team has its own entry by name. Then click the green ▶ Play button. VS Code sends the code wirelessly to that robot.
🤖
Watch, adjust, and repeat
Observe what the robot does. Fix the code in VS Code. Then click the green ▶ Play button again — every run pushes a fresh copy of your changes to the robot. Repeat until the behavior is right.
💾
Commit and push to GitHub
Saving your work to GitHub shares it with the whole team and keeps a full history of every version — nothing is ever lost.
Adding or Renaming Robots in the Run Panel

The dropdown list of robots in VS Code's Run panel is controlled by a file called launch.json, found inside the .vscode folder in your project. Each robot on your team has its own entry — called a configuration — in that file. Adding a new robot or renaming one is straightforward: you only need to change the robot's Bluetooth name in two places per entry.

1

Open .vscode / launch.json in VS Code. You will see one block per robot inside the "configurations" list.

2

To add a new robot, copy an existing block (from { to its closing },) and paste it below the last entry.

3

Change the robot's Bluetooth name in two places within the new block: the "name" field (what appears in the dropdown) and the --name argument in the "args" list (what is sent over Bluetooth). Both must match the name shown on the hub's screen exactly.

4

Save the file. The new robot name will appear immediately in the Run panel dropdown.

Example — Two Robots in launch.json
{ "configurations": [ { "name": "Thought Process", "type": "debugpy", "request": "launch", "noDebug": true, "module": "pybricksdev", "args": [ "run", "ble", "--name", "Thought Process", "main.py" ] }, { "name": "Bot Two", "type": "debugpy", "request": "launch", "noDebug": true, "module": "pybricksdev", "args": [ "run", "ble", "--name", "Bot Two", "main.py" ] } ] }

Red values are the only things you change per robot. Both must match the Bluetooth name shown on that hub's screen.

Glossary — Terms Used in This Guide
terminal
A text-based window where you type commands directly. On Mac, open it by searching for "Terminal." On Windows, search for "Command Prompt" or "PowerShell."
virtual environment
A private, isolated copy of Python and its tools that lives inside your project folder. It prevents conflicts with other software on your computer and keeps every coach's setup identical.
pip
Python's built-in package installer. Running pip install downloads and installs a tool or library. Think of it like an app store for Python tools.
pybricksdev
The command-line tool that connects to the SPIKE Hub over Bluetooth and sends your Python program to it. It is installed automatically when you run pip install -r requirements.txt.
interpreter
The specific version of Python that VS Code uses to read and run your code. Selecting the .venv interpreter tells VS Code to use the project's private Python, not your system's.
launch.json
A configuration file inside the .vscode folder that tells VS Code how to deploy your code to each robot. It lists each robot by name and contains the Bluetooth connection settings.
